Coverage boundary

This report currently measures recovered binary callables. Source-authored behavior corpora—including UI documents, behavior programs, scenarios and Lua, animation events, and world definitions—remain separate evidence surfaces and must also be crosswalked before complete behavior closure can be claimed. Package-specific expandable lists below replace a duplicated global sample of unmatched callables.

How to close the crosswalk

For each obligation, inspect the C++ oracle, source data, and live behavior, then normalize related callables into one semantic contract: trigger, conditions, state read, state written, timing/order, randomness, and player-visible outcome. Link that contract to the exact Bevy systems/components/messages that implement it. Helper callables may be marked support only with a recorded reason. Behavioral verification remains a separate final status.

Regenerate with python tools/build-behavior-crosswalk.py; verify determinism and references with python tools/audit-behavior-crosswalk.py.
